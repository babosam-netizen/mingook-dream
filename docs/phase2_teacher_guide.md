> 마지막 갱신: 2026-05-14 / v1.2.69

> 웹앱의 현재 기능 상태와 각 단계에서 주의할 점을 함께 담았습니다.

## 단계별 상세 안내

---

### ① 후보 캠프 구성 및 공약 개발용 자료 수집
**워크플로 스텝**: `prep`

**학생 화면**: 공약 개발 자료실 (LinkBoard) — 기사·영상 링크 제출 및 모아보기

**교사 할 일**:
- 교사 대시보드 → 두 번째 여정 → `[① 후보 캠프 구성·자료 수집]` 클릭
- 학생들에게 *최우선 과제를 해결할 공약*을 만들기 위한 자료를 모으도록 안내

**현재 앱 상태** ✅:
- LinkSubmit: 링크 제출 폼 (뉴스·영상 선택)
- LinkBoard: 제출된 링크 전체 목록 표시
- 사진 업로드 없음 (Canva 링크로 대체)

**주의 사항**:
- 이 단계에서는 후보 등록 UI가 아직 안 보임. 자료 수집에만 집중.
- 다음 단계(`register`)로 넘기면 후보 등록 폼이 활성화됨.

---

### ② 후보 등록 또는 선거 기자 등록 및 지지 선언문 작성
**워크플로 스텝**: `register`

**학생 화면**: 우리 모둠 후보 캠프 등록 폼 또는 선거 기자단 등록 + 지지 선언문 작성

**등록 항목**:
- 후보자 번호·닉네임 (자동 입력)
- 출마 선언 (자유 문장)
- 선거 포스터 — **Canva 임베드 링크** (이미지 업로드 없음)
- 공약 3가지
- 정책 카드뉴스 — Canva 임베드 링크
- 지지 선언문 (모둠원 각자 작성)

**선거 기자단 (후보 미등록 모둠) 활동 안내**:
- **생생한 취재**: 이름 가리기를 활용하여 여러 후보 캠프 활동에 대한 생생한 취재 및 5W1H 구조의 리포트 제공
- **빠른 정보 전달**: 선거 관련 주제에 대해 쉽고 빠르고 정확한 정보 전달 제공
- **기자 직업 윤리**: 여러 사람에게 두루 도움이 되는 기사 작성

**교사 할 일**:
- 워크플로 → `[② 후보 등록 및 지지 선언문 작성]` 이동
- 캔바 사용 방법 안내: [공유] → [임베드(</>) 코드 복사] → 앱에 붙여넣기
- 후보를 내지 않는 모둠은 명패에서 '없음'을 선택하여 선거 기자단으로 등록하도록 안내

**현재 앱 상태** ✅:
- 후보 등록 완료 시 수정 버튼으로 재편집 가능
- 등록하면 하단에 실시간 미리보기 표시
- 등록 폼에 로컬 임시저장(💾) 기능 있음

**주의 사항**:
- Canva 링크는 반드시 `view?embed` 형태의 임베드 링크 사용 (일반 공유 링크 ❌)
- Canva HTML 코드 전체를 붙여넣으면 자동으로 src 주소만 추출됨 ✅
- 모든 모둠이 등록했는지 교사 모니터링에서 확인 후 다음 단계로 이동

---

### ③ 후보 비교 및 아고라 찬반 의견
**워크플로 스텝**: `agora`

**학생 화면**: 후보 카드 전체 목록 — **투표 버튼 없음**

**교사 할 일**:
- 워크플로 → `[③ 후보 비교 및 아고라 찬반 의견]` 이동
- 학생들이 후보 카드를 읽고 각 후보의 아고라 탭에 찬반 의견을 남기도록 안내

**현재 앱 상태** ✅:
- 아고라 찬반 의견: 후보 카드 하단 `🏛️ 후보 비교 아고라` 섹션
- 투표 버튼 완전 숨김 (previewMode=true)
- 학생 화면 상단에 "⚠️ 이 단계에서는 투표하지 않습니다" 경고 문구 표시

**주의 사항**:
- 아고라 의견은 *후보에 대한 찬반*이지 *최우선 과제에 대한 찬반*이 아님. 사전 안내 필요.
- 의견 수가 어느 정도 쌓인 것을 확인한 후 다음 단계로 이동.

---

### ④ 사전 선거 여론조사
**워크플로 스텝**: `prepoll`

**학생 화면**: 사전 여론조사 응답 폼 (PollFeed)

**교사 할 일**:
1. **여론조사 먼저 만들어야 함**: 교사 대시보드 → ⚡ 여론조사 → [두 번째 여정 슬롯] → `선거 사전 여론조사 2` 활성화
2. 이유 쓰기 허용 여부를 토글에서 결정 (`✍️ 참여 시 이유 쓰기 허용`)
3. 워크플로 → `[④ 사전 선거 여론조사]` 이동
4. 여론조사가 충분히 응답되면 [📢 게시]하여 결과 공개

**현재 앱 상태** ✅:
- 이유 쓰기 기능 구현됨 (allowReason 토글)
- 결과 화면에서 각 항목의 득표율 + 이유 목록 함께 표시
- "아직 투표가 아닙니다" 안내 문구 표시

**주의 사항**:
- 여론조사를 먼저 생성하지 않으면 학생 화면에 아무것도 안 보임.
- `plannedOnly` 필터 사용 중: 해당 스텝(`prepoll`)에 연결된 여론조사만 표시됨.
- 여론조사 생성 시 `두 번째 여정 / prepoll 슬롯`을 선택해야 이 화면에 나타남.

---

### ⑤ 토론 준비 — 1·2위 후보 중심
**워크플로 스텝**: `debatePrep`

**학생 화면**: 1·2위 후보 카드(배지 강조) + 지지모둠·평가단 역할 안내 박스

**교사 할 일**:
1. 토론 도구 → [토론 열기] → 1위·2위 후보 지지 모둠과 평가단 배정
2. 토론 준비 카드(주장·근거·반론대비) 작성 안내
3. 워크플로 → `[⑤ 토론 준비]` 이동

**현재 앱 상태** ✅:
- 여론조사 1·2위 데이터 기반으로 `🥇 1위 / 🥈 2위` 배지 자동 표시
- 역할 안내 박스: "지지 모둠 / 평가단" 설명 문구
- DebateTool(우하단 플로팅)에서 토론 준비 카드 작성 가능
- 여론조사 미집계 시 "사전 여론조사 결과가 집계되면..." 안내 표시

**주의 사항**:
- 이 단계는 **여론조사 결과가 집계된 상태**여야 1·2위 카드가 표시됨.
  - 여론조사를 먼저 [📢 게시] 또는 [🔒 종료]해야 ranks가 계산됨.
- DebateTool 토론 세션이 열려 있어야 학생이 준비 카드를 작성할 수 있음.
- 모둠별 토론 역할은 **교사가 직접 DebateTool에서 배정**해야 함 (자동 배정 없음).

---

### ⑥ 토론 결과 평가 및 기사 작성
**워크플로 스텝**: `debateEval`

**학생 화면**: 기사 작성 폼 (ArticleSection)

**교사 할 일**:
1. DebateTimer를 통해 실제 토론 진행 (의장이 단계 조작)
2. 토론 후 SpeechEval(발언 평가)·StancePoll(사후 여론조사) 열기
3. 워크플로 → `[⑥ 토론 결과 평가 및 기사 작성]` 이동

**현재 앱 상태** ✅:
- ArticleSection: 기사 작성 폼 + 이미 작성된 기사 목록
- 토론 평가(발언 평가, 전후 의견 비교)는 DebateTool 패널(우하단 플로팅)에서 진행

**주의 사항**:
- 기사는 교사 승인 후 여론판에 게시됨 (또는 `autoApproveArticles: true` 시 자동 게시).
- 이 단계에서 아고라 찬반 섹션은 숨겨짐 → 학생들이 기사 작성에 집중할 수 있음.

---

### ⑦ 투표 시작 및 결과 확인
**워크플로 스텝**: `vote`

**학생 화면**: 투표 버튼 활성 후보 카드 + 선거 결과 패널

**교사 할 일**:
- 워크플로 → `[⑦ 투표 시작 및 결과 확인]` 이동
- 학생들에게 "보통·평등·직접·비밀선거의 원칙"을 구두로 한 번 더 강조
- 전원 투표 완료 확인 → 결과 패널에서 순위 확인

**현재 앱 상태** ✅:
- 투표 모달에 "보통, 평등, 직접, 비밀선거의 원칙에 따라 투표해주세요." 문구 표시
- `vote` 스텝에서만 투표 버튼 활성 (다른 스텝에서는 previewMode)
- 투표 완료 즉시 실시간으로 결과 패널에 반영

**주의 사항**:
- 비밀선거: 학생들이 서로 투표 결과를 보여주지 않도록 교실 분위기 조성 필요.
- 결과 공개 시점은 교사가 조율 (전원 투표 완료 후 결과 확인 허용).
- 가중 투표권(베네핏 카드)은 현재 비활성. 필요 시 `config` 설정으로 활성화 가능.

### ⑧ 기사 작성 — 선거 결과
**워크플로 스텝**: `finalNews`

**학생 화면**: 기사 작성 폼 + 선거 결과 패널

**교사 할 일**:
- 워크플로 → `[⑧ 선거 결과 기사 작성]` 이동
- 학생들이 선거 결과를 바탕으로 *당선 결과 기사*를 쓰도록 안내

**현재 앱 상태** ✅:
- ArticleSection: 기사 작성 폼
- ElectionResultPanel: 득표 순위 확인 가능

**주의 사항**:
- 기사 대상(target)을 '일반'이나 '선거 결과'로 설정하도록 안내.
- 기사 작성 후 교사 승인 큐에서 승인해야 여론판에 게시됨.

### ⑨ 다음 여정 안내 — 직책 및 순서
**워크플로 스텝**: `nextJourney`

**학생 화면**: 선거 결과 패널 + 세 번째 여정(국정 포털) 안내

**교사 할 일**:
- 워크플로 → `[⑨ 다음 여정 안내]` 이동
- 대통령 당선 모둠과 나머지 모둠의 직책을 안내
- Phase 3 부서 배치 설정 (교사 대시보드 → [학급 설정] → Phase 3 부서 배치)

**현재 앱 상태** ✅:
- 국정 포털 세 여정(입법부·행정부·사법부) 안내 텍스트 표시
- 이후 교사가 Phase 3으로 페이즈 전환

**주의 사항**:
- Phase 3 부서 배치를 **미리 설정**해 두어야 학생들이 세 번째 여정 진입 시 자신의 소속 부서를 바로 확인할 수 있음.
- 당선 모둠 → 행정부(또는 입법부 발의 모둠)로 배치하는 것이 일반적인 시나리오.

## 교사 대시보드 빠른 참조

### 워크플로 조작 위치
`교사 대시보드` → `📋 워크플로 가이드` → `두 번째 여정` → 단계 선택 → `[▶ 이 단계로]`

### 여론조사 생성 위치
`RoomBar` 상단 ⚡ 버튼 → [두 번째 여정 슬롯] → `선거 사전 여론조사 2` 활성화
- 이유 쓰기 허용: `✍️ 참여 시 이유 쓰기 허용` 체크박스 ON/OFF
- 결과 공개: `[📢 게시]` 버튼

### 토론 도구 제어 위치
`교사 대시보드` → `🗣️ 토론 도구` → [토론 열기] → 의장 지정, 찬반/평가단 배정

### 기사 승인 위치
`교사 대시보드` → `📋 승인 큐` → 기사 탭 → 승인/반려

| 기능 | 상태 | 비고 |
|---|---|---|
| 베네핏 카드 (가중 투표권) | 데이터 구조만 있음 | config 설정으로 활성화 가능하나 UI 미완성 |

| 당선증 PDF 출력 | 없음 | 수동으로 캔바 등에서 제작 권장 |

## 수업 흐름 팁

1. **자료수집(①)과 후보등록(②)은 같은 6차시**에 진행 가능 — 자료 수집 후 바로 등록.
2. **아고라(③)에서 토론을 충분히** 한 후 여론조사(④)를 열면 더 풍부한 이유가 나옴.
