- **두 모드 차이**: 역할중심 = 학생이 직접 검사·변호·증인이 되어 증거·증인으로 논고를 쓰고 실제 재판(기존 7단계 그대로). 판결중심 = AI 대본 재판을 참관하고 **전 모둠이 판사가 되어 판결문 작성·비교**.
- **판결중심 7단계**: ① 준비(사건개요+연기 3팀 지정) → ② 쟁점 파악 → ③ 재판 보기(대본 연기, 연기 3팀만 자기 대사 노출) → ④ 판결문 작성·게시(전 모둠) → ⑤ 온라인 토의(판결문 비교) → ⑥ 기사 → ⑦ 여론조사. 기사·여론조사는 `article3`/`poll4` 재사용(라벨만 판결중심용).
- **신규 데이터**: 사건 JSON `trialScript`(speaker별 대사 배열), RTDB `judicialIssues/{caseId}/{groupId}`(쟁점 메모). 모둠별 판결문은 기존 `verdicts/{caseId}` 재사용(`judgeGroupId` 키).
- **핵심 파일**: `JudicialVerdictTab.jsx`(신규 학생 화면), `JudicialTab.jsx`(workMode 래퍼 분기 — 기존 흐름은 `JudicialRoleTab`으로 보존), `PhaseWorkflow.getPhaseSteps()`, `Phase3JudicialQuickPanel`(verdict 분기 + 준비단계 빠른제어 강화), `BranchConfigEditor`(토글 라벨), `JudicialCaseSetupPanel`(verdict 시 AI 프롬프트에 trialScript).
- **미검증**: 라이브 방(demo24) 인터랙티브 검증은 미수행 — `docs/task.md` 참조.

## 2. 현재 운영 상태
전체 MVP는 실사용 준비 상태이며, 최근 작업은 Phase 3 사법부 워크플로 고도화(6단계 세분화 및 교사 빠른 제어 패널 신설) 및 행정부 역할중심 모드 개편(2컬럼 레이아웃 및 2단계 간소화)이다.

### 주요 구현 영역
- **Phase 1 시민광장**: 국민청원, 시민단체 결성, 캠페인 포스터/주장글/영상, 댓글·다축 평가, 여론조사, 기사 작성, 최우선 과제 확정.
- **Phase 2 선거**: 후보 캠프 자료수집, 후보 등록, 지지 선언문, 후보별 아고라 찬반 의견, 사전 여론조사, 토론 준비/평가, 본 투표, 선거 결과 기사.
- **Phase 3 국정포털**:
  - **입법부**: 준비 → 발의 → 토의 및 평가 → 상정 토론 → 표결 → 발표.
  - **행정부**: 준비 → 정책 및 예산 초안 → 예산 초안 검토 → 토의 및 평가 → 다자간 토론(국무회의) → 정책 및 예산안 최종 수정 → 최종 발표.
  - **사법부**: ① 준비·역할 배정 → ② 주장·변론서 작성 → ③ 증거 제출·검토 → ④ 모의재판 진행(토론) → ⑤ 최종 변론·구형 → ⑥ 배심원 평결·선고. 교사용 빠른 제어 패널(Pattern C) 및 학생용 게이팅·자동 스크롤 완비.
- **여론판**: 기사 승인, 관점 필터, 여론조사 결과, 기사 목록.
- **정리글 벽**: 성찰 글 작성, 색상, 비공개, 공감 이모지, 갤러리 워크.
- **토론도구**: 토론 전 준비카드, 입장 조사, 토론 중 타이머와 대본, 토론 후 평가.

- 기준 문서 `docs/debate_eval_architecture.md`에 `evalExposureEnabled`, 종료 조건, 점검 절차를 추가했다. 다음 작업자는 이 문서를 먼저 확인해야 한다.
