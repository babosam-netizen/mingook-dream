
# 행정부 역할중심 모드 워크플로우 통합 개편 문서 (v1.0)
마지막 업데이트: 2026-05-24

## 1. 개편 배경 및 목적
기존 행정부의 역할중심 모드는 학생 화면에서 1단계 "메모 제출"과 2단계 "섹션 초안 작성"이 분리되어 있어, 역할별 할일(todo)이 파편화되고 직관적이지 않았습니다. 또한 총괄 검토원(장관)이 부서원의 결과물을 모아 최종 편집하는 기능이 미비하여, 원활한 협업과 결과물 병합에 병목이 발생했습니다. 

이에 역할중심 모드의 작성 단계를 단일화된 "미션 중심의 에디터"로 통합하고, 모든 역할의 결과물을 총괄 검토원이 온전하게 취합/수정할 수 있도록 워크플로우를 개편하였습니다.

## 2. 주요 개편 사항 (Workflow & Architecture)

### 2.1 단일화된 미션 중심 편집기 (ExecutiveSectionEditor)
* **이전 방식**: 1단계(자료 조사 및 메모) -> [메모 제출] -> 2단계(역할별 섹션 초안 작성 및 예산 편성) -> [초안 제출] 의 두 단계 게이팅 구조.
* **개편 방식**:
  - `ExecutiveSectionEditor` 내에서 **역할별 미션(할 일)과 초안 작성, 예산 편성을 한 번에** 수행합니다.
  - 학생 화면 좌측에는 부여받은 미션(가이드)과 예시가 노출되며, 우측에서 텍스트(시행령, 근거, 눈높이 반영 등 역할별 텍스트)와 필요 예산 항목을 작성합니다.
  - 별도의 "메모 제출" 과정 없이 작성 후 바로 [임무 완료 및 초안 저장] 버튼을 통해 서버(`branchDrafts/{unitId}/sections/{sectionKey}`)에 저장합니다.
  - (※ 이전 메모 데이터 연동을 위한 [내 메모 가져오기] 버튼 별도 추가 제공)

### 2.2 총괄 검토원(장관)의 최종 조립 및 권한 강화 (ExecutiveFinalAssembler)
* **이전 방식**: 각 부서원이 작성한 내용이 텍스트 형태로만 보여지거나 최종 템플릿에 수동 복사해야 했으며, 예산 등 구조화된 데이터 취합이 불완전했습니다.
* **개편 방식**:
  - 총괄 검토원은 `ExecutiveFinalAssembler` 뷰에서 **[모둠원 초안 불러와 병합하기]** 기능을 사용합니다.
  - **텍스트 병합**: 각 역할별로 입력된 `policyFields`(시행령, 필요근거, 여론 눈높이 등)의 조각난 필드들이 최종안(`allFields`)으로 누락 없이 자동 병합됩니다.
  - **예산 병합**: 모든 역할에서 청구된 예산 항목(`budgetItems`)을 중복 없이 ID 기준으로 배열하여 합산(`allBudgets`)합니다.
  - **편집 권한**: 병합된 내용을 바탕으로 총괄 검토원이 텍스트 필드를 직접 수정하거나, 예산 계산기를 다시 열어 예산 항목을 추가/변경(100억 총액 조율)할 수 있도록 "완전한 편집 권한"을 부여했습니다.

### 2.3 교사 빠른 제어(Quick Panel) 연동 (Phase3ExecutiveQuickPanel)
* 학생들의 통합 워크플로우 현황을 교사가 한눈에 모니터링하고 제어할 수 있도록 개편.
* 모둠 및 부처별로 4인 역할의 **텍스트/예산 작성(isSecDone, isSecWriting)** 여부를 가로(inline) 배지 형태로 콤팩트하게 표시하여 스크롤 없이 전체 진행 상황을 직관적으로 확인 가능.
* 교사가 전체 과정을 락킹(Lock)할 수 있는 기능을 제공하며, 잠금 시 학생 화면의 `RoleSelfSelector`와 에디터 입력이 연동되어 보호 상태(`isLocked`)에 들어갑니다.

## 3. 데이터 구조 (Schema Changes)

역할별 초안 데이터가 `branchDrafts/{unitId}/sections/{sectionKey}/content` 내부에 다음의 형태로 통일되어 저장됩니다.

```json
{
  "policyFields": {
    "ordinance": "...",
    "evidence": "...",
    "discussionReflection": "...",
    "expectedEffect": "...",
    "finalScale": "..."
  },
  "budgetItems": [
    {
      "id": "...",
      "title": "...",
      "amount": 10,
      "calc": { ... }
    }
  ],
  "status": "ready"
}
```

## 4. 확인된 이슈 및 안정화 결과
- [x] **역할 잠금/배정 완료 오류 방어**: 역할을 배정했음에도 학생 에디터가 비활성화되던 버그는 메모/초안 단계 분리로 인한 로직 꼬임이었으며, 단일 에디터 렌더링 도입으로 해결 및 `isLocked` 동기화 완료.
- [x] **예산 계산기 동기화 핫픽스**: 계산기로 입력한 결과값이 목록에 유지되지 않던 버그를 `useRef(isLoaded)` 플래그를 통해 불필요한 RTDB 재동기화(덮어쓰기) 방지로 해결.
- [x] **결과**: 현재 역할중심 모드의 미션 수행 및 총괄 취합->제출까지의 데이터 흐름이 정상 동작함을 확인.
