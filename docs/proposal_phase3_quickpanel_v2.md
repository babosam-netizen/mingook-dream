# Phase 3 QuickPanel 재설계 제안 — 패턴 C 적용

**작성일:** 2026-05-14
**작성자:** Claude Code (사용자와 협의)
**상태:** 결정 확정 (입법부 먼저 적용 → 검증 후 행정·사법 확장)
**우선 적용:** 입법부 (`Phase3LegislativeQuickPanel.jsx`)

---

## 1. 배경

세 번째 여정(국정 포털) 교사 조정 화면은 현재 **3계층 구조**:
1. 사이드바 `PhaseWorkflow` — "입법부 활동 진행" 그룹 한 줄
2. 입법부 탭 진입 시 `Phase3LegislativeQuickPanel` 패널이 펼쳐짐
3. 그 안에서 stage 1~5 진행 및 액션 버튼 실행

문제점:
- "탭의 상태바에서 다음 단계 누르세요" 안내문으로 시선 이동 강제
- 액션 버튼(정식 상정·토론 시작·전광판·표결 마감)이 어느 stage에 속하는지 시각적 묶음이 약함
- 이전/다음 단계 버튼이 "어디로 가는지" 명시하지 않음

---

## 2. 결정 사항

### 2.1 사이드바 구조 — **현재 유지**
- 입법/행정/사법 그룹 한 줄로 묶어 보여주는 현재 구조 유지
- 그룹 행 선택 시 부서 탭으로 이동 → QuickPanel이 펼쳐지는 흐름 그대로

이유:
- 표결 단계는 액션이 3~4개 동시 필요(상정·토론·전광판·표결 마감) → 평탄 스텝퍼에 풀어놓으면 행 폭증
- 평탄화로 얻는 이득보다 QuickPanel 안을 정리하는 게 효과 큼

### 2.2 QuickPanel 디자인 — **패턴 C (절충형)**
- 상단: **가로 progress bar** (●━━●━━◉━━○━━○ 형태, 현재 위치 시각적 강조)
- 중간: **현재 단계 카드** — 단계명·차시·핵심 정보(1등 법안 등)
- 하단: **액션 버튼 3개 그리드** (큰 버튼, 아이콘+라벨+보조설명 세로 배치)
- 맨 아래: **이전/다음 단계 버튼** — 이동할 단계 이름 명시 (예: "이전 단계 (②토의)", "다음 단계 (④표결) ▶")

## 3. 추가 보완사항 — 입법부 ② 온라인 토의 ↔ ③ 상정

### 3.1 현재 동작 (코드 확인)
- 법안은 `_score.overall` 기준 정렬되어 정렬 결과로 표시
- 어느 법안이든 [✓ 정식 상정] 버튼 클릭 가능 (i === 0 일 때 "(추천)" 라벨)
- 추천된 법안(`teacherRecommended`)·상정 요청된 법안 우선 정렬

### 3.2 보완 요청
1. **전체 순위 명시 표시** — 1등, 2등, 3등, 4등, ... 모든 법안 등수를 카드에 명확히 표시 (현재는 점수만 보이고 등수가 암묵적)
2. **자동 추천 + 교사 선택 병행** — 1등이 기본 추천으로 강조되되, 교사가 다른 등수를 골라 상정할 수 있는 선택지 명확화
3. **단일 상정 흐름 + 부결 시 다음 법안 심사** — 한 번에 한 법안만 정식 상정·표결. 부결되면 다음 후보 법안으로 넘어감.
   - 표결 결과 화면(부결)에서 **"다음 후보 법안 심사하기"** 액션 제공
   - 클릭 시 ②토의 단계의 남은 법안 순위 카드로 복귀, 다음 등수(예: 2등) 자동 추천
   - 이미 부결된 법안은 카드에 "❌ 부결됨" 배지로 표시, 다시 상정 불가 (재상정 허용 여부는 결정 필요)

### 3.3 구현 방향
- 법안 카드에 **순위 배지**(🥇 1등 / 🥈 2등 / 🥉 3등 / 4등) 추가
- 상정 버튼 라벨: `✓ 1등 법안 정식 상정 (추천)` / `✓ 이 법안(2등) 정식 상정` 등 등수 명시
- 정렬은 점수 내림차순 유지, 동점 처리 룰 명시 필요 (현재는 정의 안 됨 — 확인 필요)

## 5. 입법부 적용 후 검토 포인트
실제 사용해본 뒤 다음 항목을 다시 논의:
1. 가로 progress bar의 클릭 동작 — 클릭으로 stage 점프 허용할지, 표시 전용으로 둘지
2. 액션 버튼 그리드의 모바일·태블릿 대응
3. 1등이 아닌 법안 상정 시 사용자 흐름 확인
4. 동점·평가 미달 케이스 처리
5. **부결된 법안 재상정 허용 여부** — 토론 결과 부결된 법안을 수정 후 재상정 가능하게 할지, 영구 차단할지
6. **부결 → 다음 법안 흐름의 stage 처리** — ④표결에서 부결 시 ②토의로 되돌아가는지, 별도 "재심사" stage를 두는지
7. 행정·사법으로 일관 적용 시 부서별 컬러 차이 (입법=파랑/행정=녹/사법=빨)
