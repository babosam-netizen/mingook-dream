## 1. 배경 / 문제

현재 역할중심 모드의 대통령실은 4명이 각각 **대통령 / 비서실장 / 민정비서관 / 경제비서관**을 맡아
"국정방향 수립·여론분석·예산검증·국무회의 준비"라는 **추상적 조율·심의** 업무를 한다.

- 6학년에게 손에 잡히는 산출물이 없어 어렵고, 일반 부처(시행령·예산)와 활동 성격이 너무 달라 복잡하다.
- 일반 부처와 "역할/할일 작성내용이 비슷해 보인다"는 혼선도 있었다(별개 라우팅 버그는 v1.3.1에서 수정).

## 2. 새 설계 한 줄 정의

> 대통령실은 **자기 공약 중 이번 통과 법령과 엮을 수 있는 공약 1개를 골라**, 그 공약을 실현하기 위한
> **시행령 + 예산 + 각 부처 업무지시**를 작성하고, 국무회의에서 이를 의장으로서 주재한다.

예시: 통과 법안 = "전쟁 화해 민간단체 지원법" → 공약 = "사회적 약자 한시적 시급 인상"
→ 그 공약을 법령 취지(갈등 해소)에 맞춰 시행령으로 구체화 + 예산 편성 + 부처별 업무지시.

## 3. 확정된 설계 결정 (사용자 선택)

| 항목 | 결정 |
|---|---|
| 역할 구성 | **4역할 재배치** (아래 4-1) |
| 공약 예산 처리 | **총액 포함 '예약분'** — 정부 총예산 안에서 대통령 몫을 먼저 잠그고, 부처는 잔여분에서 조정 |
| 공약 선택 주체 | **대통령 모둠이 선택** (교사는 빠른제어에서 덮어쓰기 가능) |

## 4. 상세 설계

### 4-1. 역할 재배치 (`DEFAULT_ROLES.executive_president` 교체)

공약 1개의 "시행령·예산·업무지시"에 모든 역할을 정렬한다.

| 역할 | 담당 | 핵심 산출물 |
|---|---|---|
| 🦅 대통령 (대표) | 공약의 중요성·우선순위 설명 + **각 부처 업무지시 정리** + 국무회의 대본 | 업무지시 목록, 국무회의 모두발언 |
| 💼 비서실장 | 공약 시행령 **조문 작성** (장관 역할) | 시행령 본문(공약 목적·시행방법) |
| 📊 경제비서관 | 공약 시행 **예산 편성** | 예산 항목·산출근거 |
| 💬 민정비서관 | 공약 **필요 근거·국민 설득 논리** | 목적·기대효과·국민 설득 포인트 |

> 4-5인 모둠 대응: 기존 `roles.slice(0, max(4, memberCount))` 패턴 유지. 5인째는 "공약 홍보원" 등 확장.

### 4-2. 새 워크플로 (역할 배정 *앞에* 공약 선택)

```
① 자료조사(기존) 
② ★공약 선택★ (신규) — 등록 공약 3개 중 1개 선택 + "이 법령과 어떻게 연결되는가" 한 줄
③ 국정 역할 배정 (기존 위치, ② 다음으로 이동)
④ 공약 시행령·예산·업무지시 작성
⑤ 국무회의 (대본 자동생성 → 함께 수정 → 예산 조정 + 업무지시)
⑥ 최종 발표
```

- 공약 데이터는 신규 입력 없음: 선거 등록분 `candidates/{presidentGroupId}.pledges`(배열)를 그대로 불러와 선택.
- 선택값 저장: `branchDrafts/exe-president/selectedPledge { index, text, lawLink }` (신규 경로).

### 4-3. 대통령실 전용 시행령 양식 (일반 부처와 다름)

일반 부처: 제1조 목적 ~ 제5조 점검.
대통령실 시행령 = **공약 시행 + 부처 업무지시 결합형**:

```
[공약 시행령]
1. 공약명 / 연계 법령
2. 목적·필요성 (왜 이 공약을 이 법령과 함께? — 민정)
3. 시행 방법·조문 (비서실장)
4. 소요 예산 (경제) — 정부 총예산의 '예약분'으로 표시
5. ★부처별 업무지시★ (대통령) — 각 부처(장관)에게 내리는 지시 N개
```

- 5번 "부처별 업무지시"는 일반 부처에 없는 신규 섹션 스키마.
- 저장: 기존 `policies/{presidentGroupId}` 재사용 + `directives[]`(부처별 지시) 필드 추가.

### 4-4. 국무회의 대본 자동 생성 + 공동 수정

- 대통령실의 선택 공약 + 시행령 + 업무지시를 템플릿에 끼워 **국무회의 대본 초안 자동 생성**.
- 생성된 대본을 토론도구 대본editor(`debateSessions/{id}/scripts/{presidentSide}.body`)에 prefill → 대통령 모둠이 **함께 수정**.
- 대본 골자: ① 대통령 모두발언(공약 의의) ② 부처별 업무지시 낭독 ③ 예산 조정 안내(공약 예약분 제외 후 잔여분 조정 요청).

### 4-5. 국무회의 순서 / 예산 조정

순서: **① 법안별 시행령·예산 조정(각 부처) → ② 대통령 공약 실천 업무지시**.

예산 조정 로직: 총예산(예 100억)에서 **대통령 공약 예약분을 먼저 차감** → 부처는 `(총액 − 예약분)` 안에서 조정.
- `Phase3ExecutiveQuickPanel`의 초과액/잔여액 계산을 "예약분 반영"으로 보정 필요.

## 6. 리스크 / 주의

- **간소화가 목적인데 산출물이 늘 위험** → 공약 1개로 범위를 좁히고, 시행령·예산·업무지시를 **하나의 통합 문서**로 묶고, 대본은 자동생성 후 살짝 수정만. 이 원칙 유지.
- **비서실장 = "장관"** 명칭 혼선 → 안내문구로 "대통령실 소속, 공약 시행령 담당"임을 명확히.
- **공동작업 모드**의 대통령실은 기존처럼 "별도 시행령 없음" 유지할지, 새 모델 적용할지 결정 필요(본 제안은 **역할중심 모드 우선** 적용, 공동작업은 추후).
- 기존 방(진행 중 수업) 데이터 호환: 구 `executive_president` 역할 잔재 방어(normalizeRole) 유지.
